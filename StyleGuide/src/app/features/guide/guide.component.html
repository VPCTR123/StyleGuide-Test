Drawers

A togglable label "Handle" that hides and shows its contents. Any content placed within the tag of the app-drawer will be shown and hidden by clicking the "handle". Classes will need to be applied to the inputs, as the app-draw by default only has the required classes for functionality.

Inputs:
displayName - Name to display on the handle
drawer_handle, drawer, icon - style clases to apply to the 3 aspects of the Draw component.

The Drawer emits a signal when it's toggled to be used if you need to do some things on the parent component when its shown or hidden. (Such as loading large amounts of data as needed)

Grids
Grids are used to organize parts of page into evenly spaced areas. They can be used as fixed column numbers (Split the div in halves, thirds, etc) or they can be used in fixed column widths, and always keep the content displayed in the fewest number of rows that will fit horizontally.

Small/Thin Grid (Its reactive)

This is a grid. The columns can be made thinner or wider, but it will always fit the cells of the grid in as few rows as possible, and if the view gets smaller it will move the cells onto lower rows to accommodate the shorter area. Buttons are added as an example for spacing and margins.

The Grid contains these "cards" and the cards can have differing heights to display a wide range of components, the grid will keep them alligned at the top. This one is the shortest.
